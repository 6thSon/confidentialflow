The Vercel deployment is failing because Vercel imported the 
wrong folder — it's trying to build artifacts/api-server 
instead of the frontend.

Fix this by creating a correct root-level vercel.json that 
tells Vercel exactly what to build and where to find output:

Create vercel.json in the workspace root (not inside any 
subfolder) with this exact content:

{
  "version": 2,
  "builds": [
    {
      "src": "artifacts/app/package.json",
      "use": "@vercel/static-build",
      "config": {
        "distDir": "dist/public"
      }
    }
  ],
  "routes": [
    {
      "src": "/(.*)",
      "dest": "/artifacts/app/dist/public/$1"
    }
  ],
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        {
          "key": "Cross-Origin-Opener-Policy",
          "value": "same-origin"
        },
        {
          "key": "Cross-Origin-Embedder-Policy", 
          "value": "require-corp"
        }
      ]
    }
  ]
}

Also check if artifacts/app/package.json has a "build" script.
If the build script outputs to dist/public, use that.
If it outputs to dist, change distDir to "dist".

Run: cd artifacts/app && npm run build
Check where the output files actually land:
ls artifacts/app/dist/
or
ls artifacts/app/dist/public/

Update distDir in vercel.json to match the actual output path.

Then push to GitHub:
git add vercel.json
git commit -m "fix: correct Vercel config to build frontend only"
git push origin main --force

   VITE_SEPOLIA_RPC_URL = https://eth-sepolia.g.alchemy.com/v2/xQZmDzEwuI70MbkBmC7Da
